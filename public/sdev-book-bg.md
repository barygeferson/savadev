## 1. Въведение

Добре дошли в **Книгата за sdev** — изчерпателен наръчник за програмния език sdev. sdev е модерен, изразителен език с уникално четим синтаксис (`forge`, `be`, `speak`, `conjure`, `summon`), създаден така, че кодът да изглежда като проза, без да жертва мощта си.


### 1.1 Защо sdev?

sdev е създаден, за да преодолее пропастта между *говоренето за код* и *писането на код*. Класическите езици използват алгебрични съкращения (`x = 1`, `def`, `var`). sdev ги заменя с глаголи и съществителни, които носят смисъл: `forge x be 1`. Резултатът е код, който се чете като рецепта или есе.


### 1.2 За кого е тази книга

Тази книга е за всички: пълни начинаещи, професионални програмисти, преподаватели и дизайнери на езици. Всяка глава започва меко и постепенно усложнява материала.


### 1.3 Как да използвате книгата

Прочетете я последователно първия път. Всеки пример може да бъде стартиран в **онлайн IDE** на https://s-dev.lovable.app или с офлайн интерпретатори. Пишете примерите — не само ги четете.


### 1.4 Конвенции

Кодовите блокове използват синтаксиса `sdev`. Изходът е показан в `// =>` коментари. Бележките **СЪВЕТ**, **ВНИМАНИЕ** и **ЗАДЪЛБОЧЕНО** маркират важна информация.


## 2. Първи стъпки

В тази глава ще инсталирате sdev (или ще пропуснете инсталацията, използвайки уеб IDE), ще напишете първата си програма и ще разберете компонентите на sdev проект.


### 2.1 Инсталация на sdev

Има три начина за стартиране на sdev:
1. **Уеб IDE** — без инсталация, отворете https://s-dev.lovable.app/ide.
2. **JavaScript интерпретатор** — `node sdev-interpreter.js моят-файл.sdev`.
3. **Python интерпретатор** — `python sdev-interpreter.py моят-файл.sdev`.


### 2.2 Вашата първа програма

Създайте файл `hello.sdev`:

```sdev
speak("Здравей, свят!")
```

Стартирайте го. Поздравления — вие сте sdev програмист.


### 2.3 REPL

Стартирайте REPL с `python sdev-interpreter.py` без аргументи. Въвеждайте изрази и те ще бъдат изчислявани веднага.


### 2.4 Структура на проекта

Типичен sdev проект има `main.sdev`, папка `lib/` и `README.md`. sdev е щастлив и с един файл.


### 2.5 Стартиране от IDE

В уеб IDE натиснете `Ctrl+Enter` за да стартирате активния таб. Изходният панел показва текста; App панелът показва UI прозорци.


## 3. Променливи и стойности

Всяка програма манипулира данни. В sdev създавате променливи с `forge` и ги обвързвате с `be`.


### 3.1 Създаване на променливи

```sdev
forge name be "Ада"
forge age be 36
forge alive be yes
```

Ключовата дума `forge` въвежда *ново* обвързване. Преприсвояването използва само името: `name be "Лъвлейс"`.


### 3.2 Петте примитивни типа

sdev разпознава пет примитивни типа: **число**, **низ**, **булева** (`yes`/`no`), **празно** и **колекция**.


### 3.3 Проверка на тип

Използвайте `essence(value)`, за да попитате за типа.


### 3.4 Конверсия на тип

Използвайте `morph(value, "type")` за конверсия.


### 3.5 Константи

Префикс `eternal` маркира стойност като неизменна.


### 3.6 Правила за именуване

Идентификаторите трябва да започват с буква или долна черта. Резервирани думи не могат да се използват.


## 4. Оператори и изрази

sdev поддържа обичайните аритметични, сравнителни и логически оператори, с английски думи-псевдоними.


### 4.1 Аритметични

Стандартни символи: `+`, `-`, `*`, `/`, `%`, `**`. Целочислено деление: `//`.


### 4.2 Сравнителни

`==`, `!=`, `<`, `>`, `<=`, `>=`. Псевдоними: `equals`, `differs`, `below`, `above`.


### 4.3 Логически

`and`, `or`, `not`. Гарантирана е оценка с прекъсване.


### 4.4 Низови оператори

Конкатенация с `+`, повторение с `*`.


### 4.5 Принадлежност

`in` и `not in` проверяват принадлежност.


### 4.6 Приоритет на оператори

От най-висок към най-нисък: унарен минус, `**`, мултипликативни, адитивни, сравнения, `and`, `or`.


## 5. Контрол на потока

Разклоняване и циклиране в sdev използват ключови думи на естествен език.


### 5.1 If / Else

```sdev
if score above 90 then
  speak("A")
else
  speak("Друго")
end
```


### 5.2 When

`when` е многоклонов сравнител с `is ... then` клонове.


### 5.3 While цикли

```sdev
while i below 5:
  speak(i); i be i + 1
end
```


### 5.4 Until цикли

`until` е обратното на `while`.


### 5.5 For-Each

```sdev
for item in [1,2,3]: speak(item) end
```


### 5.6 Range итерация

```sdev
for i in sequence(1, 10): speak(i) end
```


### 5.7 Прекъсване и продължаване

`escape` за break, `skip` за continue.


## 6. Функции

Функциите в sdev се декларират с `conjure` и се извикват по име с скоби.


### 6.1 Дефиниране

```sdev
conjure greet(name): speak("Здравей, " + name) end
```


### 6.2 Връщане на стойности

Използвайте `yield` (или `return`).


### 6.3 Параметри по подразбиране

`conjure greet(name be "приятел"): ... end`


### 6.4 Variadic функции

Използвайте `...rest`.


### 6.5 Анонимни функции

`forge double be (x) => x * 2`


### 6.6 Функции от висш ред

Функциите са първокласни стойности и се предават свободно.


### 6.7 Затваряния

Вътрешните функции хващат променливи от обхващащия обхват.


### 6.8 Рекурсия

sdev напълно поддържа рекурсия — например факториел.

## 9. Обектно ориентирано програмиране

sdev поддържа класове чрез `essence` и създаване на инстанции чрез `summon`.


### 9.1 Дефиниране на клас

```sdev
essence Dog:
  forge name
  conjure bark(): speak(name + " лае!") end
end
```


### 9.2 Създаване на инстанции

`forge rex be summon Dog(name: "Рекс")`


### 9.3 Наследяване

`essence Puppy extends Dog: ... end`


### 9.4 Методи и self

Методите имат достъп до полетата по име. Използвайте `me` за изричност.


### 9.5 Статични методи

Префикс `shared`.


### 9.6 Полиморфизъм

Презаписвайте методи свободно. Извиквайте родителя с `ancestor.method()`.


## 10. Обработка на грешки

Когато нещата се объркат, sdev предоставя инструменти за грациозно възстановяване.


### 10.1 Try / Catch

`attempt: ... rescue err: ... end`


### 10.2 Хвърляне на грешки

`raise "Възрастта не може да е отрицателна"`


### 10.3 Finally

`attempt: ... rescue ... finally: ... end`


### 10.4 Потребителски типове грешки

`essence ValidationError extends Error: ... end`


## 11. Конкурентност

sdev предлага async/await-стил конкурентност с `dispatch` и `await`.


### 11.1 Dispatch

`forge task be dispatch fetchData()` връща обещание.


### 11.2 Await

`forge data be await task`


### 11.3 Паралелни задачи

`await all([dispatch a(), dispatch b()])`


### 11.4 Канали

За producer/consumer — `channel()`, `send`, `receive`.

## 13. UI инструментариум

sdev включва вградена декларативна UI система. Изграждайте прозорци с бутони, входни полета, плъзгачи, таблици, табове и менюта — всичко от sdev код.


### 13.1 Първият ви прозорец

```sdev
window("Hello App", 400, 300)
  heading("Здравей, свят!", 1)
  button("Натисни ме", () => alert("Здрасти!"))
endwindow
```


### 13.2 Layout: Row & Column

`row ... endrow` за хоризонтално; `column ... endcolumn` за вертикално подреждане.


### 13.3 Входни полета

`input("name", "Вашето име")` — четете стойността с `uiget("name")`.


### 13.4 Чекбоксове и плъзгачи

`checkbox("agree", "Съгласен съм")`, `slider("vol", 0, 100, 1)`.


### 13.5 Табове и групи

`tabs ... tab("Профил") ... endtab ... endtabs`


### 13.6 Таблици

`table(["Име","Възраст"], [["Ада",36]])`


### 13.7 Менюта

`menu("File") menuitem("New", () => ...) endmenu`


### 13.8 Персонализация

Бутоните приемат вариант като трети аргумент: `default`, `primary`, `destructive`, `ghost`. Прозорците приемат ширина и височина.


### 13.9 Реактивни стойности

`uiget("име")` чете, `uiset("име", стойност)` записва. UI се пречертава автоматично.

## 20. Готварска книга с примери

Кратки, пълни програми, демонстриращи идиоматичен sdev.


### 20.1 FizzBuzz

```sdev
for i in sequence(1, 100):
  when:
    is i % 15 == 0 then speak("FizzBuzz")
    is i % 3 == 0 then speak("Fizz")
    is i % 5 == 0 then speak("Buzz")
    else speak(i)
  end
end
```


### 20.2 Брояч на думи

`forge text be readFile("essay.txt"); speak(magnitude(shatter(text, " ")))`


### 20.3 Todo приложение

```sdev
forge todos be []
window("Todos", 360, 480)
  input("new", "Какво да направя?")
  button("Добави", () => { gather(todos, uiget("new")); uiset("new", "") })
  for t in todos: label(t) end
endwindow
```


### 20.4 HTTP заявка

`forge res be await fetch(url); forge data be jsondecode(res.body)`


### 20.5 Подскачаща топка

```sdev
canvas(400, 400)
forge x be 200; forge y be 200; forge dx be 3; forge dy be 2
frame((t) => { fill("black"); rect(0,0,400,400); fill("cyan"); circle(x,y,20); x be x+dx; y be y+dy; if x<0 or x>400 then dx be -dx end; if y<0 or y>400 then dy be -dy end })
```


## 21. Често задавани въпроси

Реални въпроси от реални sdev потребители.


### 21.1 Бърз ли е sdev?

За интерпретирано изпълнение производителността е сравнима с Python. Bytecode компилаторът дава 3–5× ускорение.


### 21.2 Мога ли да го използвам в продукция?

sdev е най-подходящ за скриптове, образование и прототипи.


### 21.3 Защо не Python?

sdev съществува за тези, които искат *по-четим* алтернативен синтаксис.


### 21.4 Къде да докладвам бъгове?

Отворете issue в GitHub хранилището.


### 21.5 Мога ли да допринеса?

Да! sdev е с отворен код.


## 22. Речник

Бързи дефиниции на ключови термини.


### 22.1 А–Й

**Анонимна функция** — функция без име.
**Затваряне** — функция, която улавя променливи от лексикалния си обхват.
**Вечен** — модификатор за неизменими обвързвания.


### 22.2 К–Р

**Колекция** — списък или речник.
**Ламбда** — синоним за анонимна функция.
**Морф** — вградена за конверсия на типове.


### 22.3 С–Я

**Обект** — инстанция на клас.
**Обещание** — стойност, която ще пристигне по-късно.
**ВМ** — байткод виртуалната машина.
